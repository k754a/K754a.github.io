How I generated random dungeon terrain
K754a
May 1, 2024
            Hi! So, when it comes to making terrains and 2D levels, there are lots of different algorithms, I picked Random Walk generation.
    
            But what is a random generation algorithm?"
    
                A random walk generation algorithm is an equation that calculates the next place to generate something.
            
                An example could be: 
                        (Xn+1, Yn+1) = (Xn, Yn) + (系x, 系y)

                But what does this equation do?

                        Xn+1, Yn+1  represents the position (POS) after n+1 steps in 2d space (X, Y)

                              Xn, Yn is the POS after n steps
                               

                              (系x, 系y) is the random values, it can be (-1, -1), (-1, 0), (-1, 1) (0, -1)...

                                Then with that we move the walker to that square and generate again.

                                If we want to prevent spawning on the same tile, we check if there is a square on the tile, if there's not we generate.

How did we texture the tiles?


               The way we textured the tiles was by checking all the tiles around the first tile
               then we generated a code based off the tiles around it.
               We mached the code in a database and from that set it to the tile.
               

               



               The generated code can look like this, depending on the position:

               -0b1111

               -0b10100000

               -0b10000110

               -0b0101
